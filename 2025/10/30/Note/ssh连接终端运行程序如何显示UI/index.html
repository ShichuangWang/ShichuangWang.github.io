学习笔记--SSH连接

发表于 2025-10-30 更新于 2025-10-31
A（Windows 11)上通过 SSH 连接到B（Ubuntu 20.04 Desktop） 在 SSH 终端中运行一个有图形界面的程序（比如 Qt、Tkinter、Matplotlib GUI、Isaac Sim等），程序在 B 的显示器上不会弹出界面。

🧩 一、现象描述

  • A（Windows 11) 上通过 SSH 连接到B（Ubuntu 20.04 Desktop）
    在 SSH 终端中运行一个有图形界面的程序（比如 Qt、Tkinter、Matplotlib GUI、Isaac Sim等），程序在 B 的显示器上不会弹出界面。

🚫 二、原因分析

1️⃣ SSH 默认禁止 GUI 显示

当你通过 SSH 登录 Ubuntu 时，默认情况下：

  • SSH 会话是“无图形环境”的；

  • 它没有继承到 B 本地的 X server（显示服务器）；

  • 因此，程序虽然在 B 上运行，但它不知道该把窗口画到哪个显示器上（$DISPLAY 环境变量为空）。

表现：

GUI 程序尝试连接显示服务器失败，于是不会弹出界面。

✅ 方案 3：使用远程桌面（RDP 或 VNC）

如果你经常需要运行带UI的程序，可以考虑使用桌面远程控制：

  • xrdp：Windows 自带远程桌面客户端就能连接；

  • VNC（如 TigerVNC、RealVNC）：跨平台图形访问。

这样你可以远程进入 Ubuntu 的图形桌面，直接双击运行软件，就不会遇到 $DISPLAY 问题。
